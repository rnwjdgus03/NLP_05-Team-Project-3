## 1. 첫 정량 채점 (골드 vs 파이프라인)

`gold_measurement_final.csv`(사람이 KOSIS를 직접 조회한 정답지)로 파일럿 33 measurement를 `score_gold.py`로 채점.

| 지표 | 값 | 의미 |
|---|---|---|
| 검색 recall@5 | **62.5%** | 정답 통계표를 후보 top-5에서 찾은 비율 (37.5% 놓침) |
| 판정 정확도 | **30.4%** | 최종 verdict가 골드와 일치한 비율 |
| 판정 상한 | **57%** | 정답표를 찾은 비율. recall이 판정 천장을 만든다 |

판정 정확도가 낮은 건 "판정을 못해서"가 아니라 **검색이 표를 못 찾으면 판정도 못 맞히기 때문**. 즉 병목은 검색.

## 2. 오차 기준 재조정 (판정 로직)

- 팀 공식 기준 = **거의 정확일치(엄격)**.
- 기존 `judge()`의 **절대오차 지름길(0.5%p)**이 상대오차 큰 값을 일치로 오판했다. 예: 주장 1.4% vs 실제 1.31% → 절대 0.09%p라 "일치"로 봤지만 상대오차는 6.5%.
- **상대오차 기준으로 변경**: 일치 ≤1.5% / 판정보류 1.5~4% / 불일치 >4%.
- 효과: 판정 정확도 **21.7% → 30.4%**. 골드의 같은표 8건 중 7건 일치.
- 남은 1건(C001: 6838억 vs 6836억, 0.03%)은 뉴스(산업부 발표)와 KOSIS 출처 차이 → **팀 확인 대상**.

## 3. 검색 방식 3자 비교 — 결정적 실험

Colab에서 BGE-M3·KURE-v1 임베딩 인덱스(각 107,138표)를 구축하고, lexical과 같은 골드로 비교.

| 검색 방식 | recall@1 | recall@5 |
|---|---:|---:|
| **lexical (키워드)** | **54.2%** | **62.5%** |
| BGE-M3 hybrid | 45.8% | 58.3% |
| KURE-v1 hybrid | 45.8% | 58.3% |

- **임베딩 두 모델 모두 lexical보다 열세.** 한국어 특화 KURE도 BGE-M3와 동일(58.3%).
- **임베딩만 찾은 정답표 = 0건** (임베딩 hit ⊂ lexical hit). 합집합도 62.5%에서 안 늘어남 → **보완 가치조차 없음**.

### 왜 임베딩이 지는가

KOSIS 표 이름이 "품목별 수출액", "취업자수" 같은 **간결한 통계 용어**라, 뉴스 지표어가 표 이름에 그대로 들어있다. **키워드 겹침이 강하고 정밀한 신호**인 반면, 임베딩은 의미상 비슷한 엉뚱한 표(수출 관련 표 수십 개)를 끌어와 정밀도를 떨어뜨린다. 자연어 문장이 아니라 용어 나열이라 임베딩의 장점(유의어·의역)은 안 살고 노이즈만 는다. 또한 hybrid는 후보 점수가 조밀해져 `AMBIGUOUS_TABLE` 게이트에 대량 걸려 판정이 붕괴했다(판정 4%).

### 3-1. KOSIS 통합검색 API도 비교 (추가 실험)

임베딩이 진 뒤, KOSIS 자체 통합검색(`statisticsSearch.do`, searchNm)을 대안으로 테스트(`test_kosis_integrated_search.py`). 각 measurement의 지표어로 검색해 `gold_tbl_id`가 상위에 있는지 확인.

| 검색 방식 | recall (정답표 발견) |
|---|---:|
| **lexical** | **62.5%** |
| BGE-M3 hybrid | 58.3% |
| KURE hybrid | 58.3% |
| KOSIS 통합검색 | **8.3%** (recall@10) |

통합검색은 실적표 대신 **전망지수·물가지수·설문조사**를 상위로 노출한다(반도체→EBSI 전망, 자동차→수출물량지수, 바이오헬스→설문 1위). 화장품·바이오 등 일부 품목표는 상위 2~3위에 찾았으나 2/24로 채택 근거가 안 된다. **임베딩·통합검색 모두 lexical에 열세** — 검색 방식 4종 비교 완료.

## 3-2. 게이트·추출 채점과 scope 버그 수정

A팀이 `gold_verifiable`·`gold_measurement_correct`(109행)를 채워, 게이트·추출 정확도를 처음 측정.

| 단계 | 지표 | 값 |
|---|---|---:|
| ② 게이트 | 정밀도 | 74.4% |
| ② 게이트 | 재현율 | **90.6%** (scope만 수정 69.0%, 최초 59.5%) |
| ② 추출 | 필드 정확도 | 84.6% |

**게이트 scope 오분류 버그** — 게이트가 검증 가능한 값을 과도 반려(재현율 59.5%)했다. 원인: 한국 수출액·무역수지·선박수출이 `claim_domain_scope='해외통계·정책'`으로 오분류(HCX가 "수출=달러=해외"로 착각) → `OUT_OF_KOSIS_SCOPE`로 반려. `correct_trade_scope`(extract_hcx) + `patch_trade_scope.py`로 교정 → **재현율 59.5% → 69.0%**(정밀도 유지). 교정된 값들은 게이트를 통과했으나 무역수지 계산식·선박 코드셋 등 **다음 단계 병목**에 걸림.

**골드 v1 기준 통일 완료(2026-07-23).** `POLICY_VALUE`, `CONDITION`, `CONTEXT`인데 `gold_verifiable=Y`였던 10건을 N으로 통일하고, scope 보정 후 게이트 결과를 다시 병합했다. 동결 결과는 109행, verifiable 32건, READY 39건이며 정밀도 74.4%, 재현율 90.6%, READY 추출 필드 정확도 84.6%다. 남은 false negative 3건은 모두 `measurement_role=이전값` 제외 사례다.

동결 파일은 `outputs/gold/gold_measurement_v1_locked.csv`이며, 라벨 변경 감사 CSV·metrics JSON·동결 보고서를 함께 생성한다. 2026-07-23에 READY 39건으로 BGE Top-K → Mapping-end 재실행을 완료했다. BGE 경로의 검색 최솟값은 Top-2였지만 READY 0건, 기술 ITEM/OBJ 골드 적중 0/20이므로 자동 매핑은 보류했다. 운영 lexical 경로의 Poc → Mapping-end 재실행은 별도 과제로 남는다.

## 4. 확정된 결정

- **검색 = lexical 단독 채택.** 임베딩(BGE-M3·KURE)·Colab·리랭커는 이 도메인에서 이득 0이라 필수 경로에서 제외 (코드·인덱스는 재현·재검토용 보존).
- **판정 = 상대오차 엄격 기준** (골드 반영, 커밋 완료).
- 이 결정은 감이 아니라 **골드로 3자 엄격 비교한 데이터 기반**. 멘토의 "측정 먼저" 원칙의 교과서 사례.

## 5. 다음 병목과 해법

병목은 **검색 recall(62.5%, 37.5% 놓침)**. 임베딩·통합검색 모두 탈락했으므로 해법은 둘로 좁혀짐:

1. **키워드 확장** — lexical을 더 똑똑하게. "수출 증가율" 같은 일반어에 품목·SITC 용어를 보강해 매칭 폭 확대. (검색 자동화의 남은 여지)
2. **코드북** — 반도체(DT_127005)·정비사(DT_444002)처럼 **어떤 자동 검색도 못 찾는 obscure 실적표**는 사람 확정 매핑이 유일한 확실한 길. 자주 나오는 어려운 지표부터 코드북에 고정.

놓친 정답표 상당수가 산업부·진흥원 실태조사 등 **검색으로 닿기 어려운 표**라, 검색 자동화만으로 recall을 크게 올리긴 어렵다. **키워드 확장으로 쉬운 것을 줍고, 나머지는 코드북**이 현실적 조합.
