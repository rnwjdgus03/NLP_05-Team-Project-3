연구 계획서

AI 기반 뉴스 수치 주장 검증을 위한 KOSIS 통계표 매핑 및 추출 스키마 설계

본 문서는 뉴스 기사에 포함된 수치 기반 주장을 공식 통계 데이터와 연결하기 위한 실전 1 연구 계획서이다. 실전 1의 목표는 전체 사실검증 시스템을 완성하는 것이 아니라, 뉴스 기사와 KOSIS 통계표를 함께 분석하여 향후 자동 검증 파이프라인의 기반이 되는 추출 스키마와 통계표 매핑 구조를 설계하는 것이다.

1. Problem Definition

1.1 문제점: 비즈니스 니즈

뉴스 기사에서는 통계청, 국가데이터처, KOSIS 등 공식 통계를 인용하여 다양한 수치 기반 주장을 제시한다. 예를 들어 “2024년 과수 농가 중 65세 이상 비율은 64.2%다”, “전년 대비 출생아 수가 감소했다”, “청년 실업률이 증가했다”와 같은 문장들이 이에 해당한다.

그러나 이러한 수치 주장이 실제 공식 통계와 일치하는지 확인하려면 사람이 기사 문장을 읽고, 주장에 해당하는 지표와 시점, 단위, 모집단을 파악한 뒤, KOSIS에서 대응되는 통계표를 찾아 직접 계산해야 한다. 이 과정은 시간이 오래 걸리고, 통계표 이름과 뉴스 표현이 다르거나 단위·시점·분류 기준이 어긋나는 경우 검증이 어렵다.

따라서 본 프로젝트의 비즈니스 니즈는 다음과 같다.

뉴스 기사에 포함된 수치 기반 주장이 공식 통계와 일치하는지 판단할 수 있도록, 뉴스 주장과 KOSIS 통계표를 연결 가능한 구조로 정리하라.

1.2 Task Reformulation

초기 접근 방법은 단순히 “기사에서 숫자가 포함된 문장을 찾는 것”이다. 하지만 숫자가 포함되어 있다고 해서 모두 검증 가능한 주장은 아니다.

예를 들어 “경제 상황이 크게 악화됐다”는 수치가 없어 직접 검증하기 어렵고, “올해 농가 고령화가 심각하다”는 표현은 공식 통계와 연결하려면 시점, 대상, 지표 정의가 추가로 필요하다. 반대로 “2024년 과수 농가 중 65세 이상 비율은 64.2%”는 공식 통계표가 존재하고 계산 기준이 명확하다면 검증 가능한 주장이다.

따라서 본 프로젝트에서는 문제를 다음 두 하위 문제로 재정의한다.

뉴스 기사에서 검증 가능한 수치 기반 주장을 탐지하고, 이를 구조화된 스키마로 추출한다.

추출된 주장에 대응 가능한 KOSIS 통계표를 찾고, API 호출에 필요한 파라미터와 계산 방식을 정리한다.

즉, 본 프로젝트는 단순 문장 분류가 아니라 뉴스 주장과 공식 통계표를 연결하기 위한 Information Extraction 및 통계표 매핑 문제로 볼 수 있다.

1.4 이 문제가 중요한 이유

뉴스 사실검증에서 가장 어려운 부분은 단순히 숫자를 비교하는 것이 아니라, 기사 문장 속 표현을 공식 통계표의 구조와 맞추는 것이다.

예를 들어 기사에서는 “올해”, “지난해”, “고령 농가”, “과수 농가”, “증가율”처럼 자연어로 표현하지만, KOSIS 통계표에서는 시점, 분류 항목, 단위, 항목 ID, 기관 ID, 통계표 ID 등 구조화된 파라미터로 표현된다.

따라서 이 연결 구조를 잘 설계하면 이후 실전 2와 종합 프로젝트에서 다음 작업이 가능해진다.

자연어 주장을 KOSIS API 요청으로 변환

공식 통계 기반 자동 수치 비교

일치 / 불일치 / 판단불가 판정

판정 근거 설명 생성

검증자 리뷰 시스템 구축

2. Background & Baseline

2.1 Related Works

뉴스 사실검증과 통계 기반 검증에는 여러 접근 방식이 존재한다.

첫째, 규칙 기반 접근이 있다. 숫자, 퍼센트, 증가·감소 표현 등을 정규표현식으로 탐지하고, 특정 키워드를 기준으로 검증 가능 문장을 찾는 방식이다. 구현이 쉽고 빠르지만, 문맥 이해가 부족하고 다양한 표현을 포착하기 어렵다.

둘째, 정보 추출 기반 접근이 있다. 문장에서 지표, 시점, 단위, 값, 모집단 등을 추출하여 구조화된 형태로 변환한다. 이 방식은 공식 통계표와 연결하기에 적합하지만, 정확한 스키마 설계와 라벨링 기준이 필요하다.

셋째, 검색 및 매핑 기반 접근이 있다. 추출된 주장 정보를 바탕으로 KOSIS 통계표 후보를 검색하고, 통계표명, 분류 항목, 단위, 시점을 비교하여 매칭한다. 본 프로젝트의 B팀 역할은 이 접근에 가깝다.

넷째, LLM 기반 접근이 있다. LLM을 활용하여 기사 문장을 해석하거나 주장 구조를 추출할 수 있다. 다만 LLM만으로 공식 통계값을 판단하면 환각 가능성이 있으므로, 실제 판정에는 KOSIS API와 같은 공식 데이터 조회가 반드시 필요하다.

2.2 이번 프로젝트의 Baseline

본 실전 1의 baseline은 다음과 같다.

규칙 기반 수치 주장 탐지 + 수동 KOSIS 통계표 매핑

구체적으로는 다음 방식으로 진행한다.

기사 본문에서 숫자, 퍼센트, 증감 표현이 포함된 문장을 규칙 기반으로 추출한다.

추출된 문장을 사람이 검토하여 검증 가능 여부를 라벨링한다.

검증 가능 문장에 대해 지표, 시점, 값, 단위, 모집단을 수동으로 구조화한다.

KOSIS 통계목록 API와 통계자료 API를 활용하여 대응 가능한 통계표를 찾는다.

통계표의 분류 항목과 단위를 확인하여 검증 가능 여부를 판단한다.

이 baseline을 선택한 이유는 다음과 같다.

실전 1 기간이 짧기 때문에 완전 자동화보다 구조 설계가 중요하다.

이후 자동화 모델을 만들기 위한 기준 데이터와 스키마가 필요하다.

통계표 매핑은 도메인 이해가 필요하므로 초기에 수동 분석이 필수적이다.

대표 사례를 End-to-End로 보여주기 쉽다.

3. Proposed Method

3.1 핵심 아이디어

본 프로젝트의 핵심 아이디어는 뉴스 주장 스키마와 KOSIS 통계표 스키마를 중간 매핑 테이블로 연결하는 것이다.

뉴스 기사 쪽에서는 다음 정보를 추출한다.

region: 지역

claim_type: 비율, 증감, 비교, 규모 등

verifiable: 검증 가능 여부

table_id: 통계표 ID

available_period: 제공 시점

required_params: API 호출에 필요한 파라미터

3.4 왜 효과가 있을 것으로 예상하는가

본 방법이 효과적일 것으로 예상하는 이유는 다음과 같다.

H1. 뉴스 주장과 통계표를 각각 구조화하면, 검증 가능 여부를 더 명확히 판단할 수 있다.

H2. KOSIS API 응답의 분류 항목과 단위를 정리하면, 단순 키워드 검색보다 정확한 통계표 매핑이 가능하다.

H3. 판단불가 사례를 별도로 유형화하면, 이후 자동 검증 시스템에서 불확실성을 더 안정적으로 처리할 수 있다.

H4. 대표 검증 가능 케이스를 End-to-End로 정리하면, 실전 2의 대화형 KOSIS API 에이전트 설계에 직접 활용할 수 있다.

4. Experiment Design

4.1 수행할 실험

E1. 조선일보 기사 크롤링 및 수치 포함 문장 추출기사 검색 결과를 수집하고 본문을 전처리한 뒤, 숫자와 통계 관련 표현이 포함된 문장을 추출한다.

E2. 검증 가능 주장 후보 라벨링추출된 문장을 검증 가능, 부분 가능, 판단불가로 분류한다.

E3. 주장 추출 스키마 적용검증 가능 후보 문장에 대해 지표, 값, 단위, 시점, 모집단, 지역, 주장 유형을 추출한다.

E4. KOSIS 통계목록 API 탐색통계목록 API를 사용하여 관련 통계표 후보를 탐색한다.

E5. KOSIS 통계자료 API 호출 및 응답 구조 분석후보 통계표에 대해 실제 데이터를 조회하고, 응답 컬럼과 분류 구조를 분석한다.

E6. 뉴스 주장과 통계표 매핑A팀이 추출한 주장 후보와 B팀이 정리한 통계표를 연결하여 매핑표를 작성한다.

E7. 대표 검증 가능 케이스 분석1~3개 사례에 대해 기사 주장, KOSIS 통계표, 필요한 계산식, 판단 가능 여부를 정리한다.

8. 대표 사례 예시

기사 주장:

“2024년 과수 농가 중 65세 이상 비율은 64.2%였다.”

추출 스키마:

metric: 65세 이상 과수 농가 비율

value: 64.2

unit: %

time: 2024

population: 과수 농가

claim_type: RATIO

KOSIS 매핑:

통계표: 경영주 연령별 농가

필요한 분류: 영농형태 = 과수

필요한 연령 구간: 65~69세, 70~74세, 75~79세, 80세 이상

계산식: 65세 이상 농가 수 합계 / 전체 과수 농가 수 × 100

검증 가능 여부: 가능

판정 예시:

KOSIS 통계표에서 2024년 전체 과수 농가는 166,558가구이고, 65세 이상 구간의 합계는 106,877가구이다. 이를 계산하면 106,877 / 166,558 × 100 = 약 64.2%이므로 기사 주장은 공식 통계와 일치하는 것으로 판단할 수 있다.

9. 결론

본 실전 1의 목표는 완전한 자동 사실검증 시스템을 만드는 것이 아니라, 뉴스 기사와 KOSIS 공식 통계를 연결하기 위한 기반 구조를 설계하는 것이다.

핵심은 뉴스 문장에서 “무엇을 주장하는지”를 구조화하고, KOSIS 통계표에서 “무엇을 검증할 수 있는지”를 정리한 뒤, 두 정보를 매핑하는 것이다. 이 과정을 통해 실전 2에서는 자연어 질문을 KOSIS API 요청으로 변환하는 에이전트로 확장할 수 있고, 종합 프로젝트에서는 일치 / 불일치 / 판단불가를 자동 판정하는 End-to-End 사실검증 시스템으로 발전시킬 수 있다.
