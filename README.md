# 연구 계획서

## AI 기반 뉴스 사실검증 시스템: 공공 통계 기반 수치 주장 자동 검증

> 본 문서는 뉴스 기사에 포함된 수치 기반 주장을 공식 통계 데이터와 비교하여 사실 여부를 검증하는 AI 시스템 설계 계획서입니다.  
> 프로젝트는 실전 1, 실전 2, 종합 프로젝트로 이어지는 단계적 구조를 가지며, 최종 목표는 뉴스 주장 탐지부터 공식 통계 조회, 비교 판정, 설명 생성까지 수행하는 End-to-End 사실검증 시스템을 구축하는 것입니다.

---

## 1. Problem Definition

### 1.1 문제점: 비즈니스 니즈

뉴스 기사와 온라인 콘텐츠에서는 통계청, 국가데이터처, KOSIS 등 공식 통계를 근거로 다양한 수치 기반 주장이 제시된다. 예를 들어 “2024년 과수 농가 중 65세 이상 비율은 64.2%다”, “출생아 수가 전년 대비 감소했다”, “청년 실업률이 상승했다”와 같은 문장들이 이에 해당한다.

그러나 이러한 수치 주장이 실제 공식 통계와 일치하는지 확인하려면 사람이 기사 내용을 읽고, 주장에 포함된 지표·시점·단위·모집단을 해석한 뒤, KOSIS 등 공식 통계 시스템에서 대응되는 통계표를 찾아 직접 비교해야 한다.

이 과정에는 다음과 같은 문제가 있다.

- 뉴스 표현과 통계표 표현이 다르다.
- 기사에는 “올해”, “지난해”, “고령 농가”처럼 모호한 표현이 자주 등장한다.
- 공식 통계표는 기관 ID, 통계표 ID, 분류 항목, 단위, 수록 시점 등 구조화된 파라미터로 구성되어 있다.
- 공식 통계가 존재하더라도 기사 수치와 바로 비교되지 않고, 합계·비율·증감률 계산이 필요한 경우가 많다.
- 사람이 직접 검증하면 시간이 오래 걸리고, 검증 누락이나 판단 지연이 발생할 수 있다.

따라서 본 프로젝트의 비즈니스 니즈는 다음과 같다.

**뉴스 기사에 포함된 수치 기반 주장을 자동으로 탐지하고, 공식 통계 데이터와 비교하여 일치 / 불일치 / 판단불가를 판정할 수 있는 AI 기반 사실검증 시스템을 구축한다.**

### 1.2 Task Reformulation

초기 접근 방법은 “뉴스 기사에서 숫자가 포함된 문장을 찾아 공식 통계와 비교한다”는 단순한 방식이다. 그러나 숫자가 포함된 모든 문장이 검증 가능한 주장은 아니다.

예를 들어 “경제 상황이 크게 악화됐다”는 공식 통계와 직접 연결하기 어렵고, “농가 고령화가 심각하다”는 지표와 시점, 모집단이 명확하지 않으면 검증하기 어렵다. 반면 “2024년 과수 농가 중 65세 이상 비율은 64.2%다”는 지표, 시점, 대상, 수치가 비교적 명확하므로 공식 통계를 통해 검증할 수 있다.

따라서 본 프로젝트는 다음 하위 문제로 재정의된다.

1. 뉴스 기사에서 검증 가능한 수치 기반 주장을 탐지한다.
2. 주장에 포함된 지표, 값, 단위, 시점, 모집단, 지역 등을 추출한다.
3. 추출된 정보를 바탕으로 KOSIS 등 공식 통계표 후보를 찾는다.
4. 공식 통계 API를 호출하기 위한 파라미터를 구성한다.
5. 통계 데이터를 조회하고 필요한 계산을 수행한다.
6. 기사 주장과 공식 통계값을 비교하여 일치 / 불일치 / 판단불가를 판정한다.
7. 판정 결과와 근거를 사람이 이해할 수 있는 설명으로 생성한다.

즉, 본 프로젝트는 단순 문장 분류 문제가 아니라 **Information Extraction + Statistical Table Retrieval + Numerical Reasoning + Explainable Fact Verification** 문제로 볼 수 있다.

### 1.3 Task 설계

본 프로젝트의 최종 Task는 다음과 같다.

**공식 통계 기반 뉴스 수치 주장 자동 검증**

하위 태스크는 다음과 같다.

- 뉴스 기사 수집 및 전처리
- 수치 기반 주장 탐지
- 주장 유형 분류
- 주장 정보 추출
- KOSIS 통계표 탐색 및 매핑
- KOSIS API 요청 파라미터 구성
- 공식 통계 데이터 조회
- 표 연산 및 수치 계산
- 기사 주장과 공식 통계 비교
- 일치 / 불일치 / 판단불가 판정
- 판정 근거 설명 생성
- 검증자 리뷰 및 최종 판단 보정

최종 산출물은 다음과 같다.

- 뉴스 기사 기반 사실검증 AI 시스템 프로토타입
- 수치 기반 주장 탐지 및 추출 스키마
- KOSIS 통계표 매핑 구조
- 공식 통계 조회 및 계산 모듈
- 일치 / 불일치 / 판단불가 판정 결과
- 판정 근거 설명
- End-to-End 서비스 또는 API 형태 결과물

### 1.4 이 문제가 중요한 이유

뉴스에 포함된 통계 수치는 사회적 의사결정과 여론 형성에 큰 영향을 미친다. 하지만 통계가 잘못 인용되거나, 일부 수치만 발췌되거나, 시점과 모집단이 다르게 해석되면 독자는 왜곡된 정보를 받아들일 수 있다.

공식 통계 기반 자동 검증 시스템은 다음과 같은 가치가 있다.

- 수작업 검증의 시간과 비용을 줄일 수 있다.
- 통계 인용 오류를 빠르게 탐지할 수 있다.
- 기사 수치의 근거와 계산 과정을 투명하게 제시할 수 있다.
- 판단불가 사례도 명확히 분류하여 검증 한계를 설명할 수 있다.
- 향후 언론 모니터링, 공공데이터 활용 서비스, RAG 기반 검증 시스템으로 확장 가능하다.

## 2. Background & Baseline

### 2.1 Related Works

- **규칙 기반 수치 주장 탐지:** 숫자, 퍼센트, 증감 표현, 비교 표현 등을 정규표현식으로 탐지하는 방식이다. 구현이 쉽고 빠르지만, 문맥과 지표 의미를 깊게 이해하기 어렵다.

- **Information Extraction 기반 접근:** 문장에서 지표, 값, 단위, 시점, 모집단 등을 추출하여 구조화한다. 공식 통계와 연결하기에 적합하지만, 추출 스키마 설계와 라벨링 기준이 중요하다.

- **Table Retrieval / Semantic Matching:** 자연어 주장과 통계표명, 분류 항목, 메타정보를 비교하여 대응 가능한 통계표를 찾는 방식이다. 뉴스 표현과 통계표 표현 사이의 차이를 줄이는 것이 핵심이다.

- **RAG 기반 접근:** 통계표 요약, 메타정보, API 결과를 검색 가능한 지식베이스로 만들고, 질의에 맞는 근거 데이터를 검색해 답변한다. 다만 실제 판정에는 공식 API 조회와 수치 계산이 필요하다.

- **LLM 기반 사실검증:** LLM을 활용해 주장 구조를 추출하거나 판정 설명을 생성할 수 있다. 그러나 LLM만으로 통계 수치를 판단하면 환각 가능성이 있으므로, 공식 통계 API와 결합해야 한다.

### 2.2 이번 프로젝트의 Baseline

본 프로젝트의 baseline은 다음과 같다.

**규칙 기반 주장 탐지 + 수동 KOSIS 매핑 + 공식 통계 기반 계산 판정**

구체적인 baseline 흐름은 다음과 같다.

1. 기사 본문에서 숫자, 퍼센트, 증감 표현이 포함된 문장을 추출한다.
2. 사람이 검토하여 검증 가능 주장 후보를 선별한다.
3. 주장별로 지표, 값, 단위, 시점, 모집단을 수동 추출한다.
4. KOSIS 통계목록 API로 후보 통계표를 찾는다.
5. KOSIS 통계자료 API로 실제 수치를 조회한다.
6. 필요한 경우 합계, 비율, 증감률 등을 계산한다.
7. 기사 수치와 공식 통계값을 비교하여 판정한다.

선정 이유는 다음과 같다.

- 프로젝트 초기에 완전 자동화보다 검증 가능한 구조 설계가 중요하다.
- KOSIS 통계표의 구조와 API 파라미터를 먼저 이해해야 한다.
- 대표 사례를 End-to-End로 성공시키면 이후 자동화의 기준점이 된다.
- 실전 1, 실전 2, 종합 프로젝트로 자연스럽게 확장할 수 있다.

## 3. Proposed Method

### 3.1 핵심 아이디어

본 프로젝트의 핵심 아이디어는 **뉴스 수치 주장을 공식 통계 API 요청으로 변환 가능한 구조로 바꾸고, 조회된 통계값과 비교하여 판정하는 파이프라인**을 구축하는 것이다.

전체 구조는 다음과 같다.

```text
뉴스 기사 입력
→ 전처리 및 문장 분리
→ 검증 가능 주장 탐지
→ 주장 유형 분류
→ 핵심 정보 추출
→ KOSIS 통계표 후보 검색
→ API 요청 파라미터 구성
→ 공식 통계 조회
→ 표 연산 및 수치 계산
→ 기사 주장과 공식 통계 비교
→ 일치 / 불일치 / 판단불가 판정
→ LLM 기반 설명 생성
→ 검증자 리뷰
```

주장 추출 스키마는 다음과 같다.

```text
claim_text: 원문 주장
claim_type: 비율 / 증감 / 규모 / 비교 / 전망
metric: 검증하려는 지표
value: 기사에 나온 수치
unit: %, 명, 가구, 원 등
time: 기준 시점
population: 대상 집단
region: 지역
context: 앞뒤 문맥
verifiable: 검증 가능 여부
```

KOSIS 통계표 스키마는 다음과 같다.

```text
org_id: 기관 ID
table_id: 통계표 ID
table_name: 통계표명
period_type: 연 / 월 / 분기
unit: 단위
dimensions: 분류 항목
items: 조회 항목
available_period: 제공 시점
required_params: API 호출 파라미터
calculation_needed: 계산 필요 여부
```

### 3.2 Baseline 대비 무엇이 다른가

| 구분 | Baseline | Proposed |
|---|---|---|
| 주장 탐지 | 숫자 포함 문장 추출 | 검증 가능 주장 탐지 및 유형 분류 |
| 정보 추출 | 사람이 문장 단위 확인 | 지표·값·단위·시점·모집단 스키마화 |
| 통계표 탐색 | 사람이 직접 검색 | KOSIS API와 통계표 요약 기반 후보 탐색 |
| 공식 데이터 조회 | 수동 조회 | API 파라미터 구성 후 자동 조회 |
| 판정 | 사람이 계산 및 판단 | 수치 비교 로직 기반 일치/불일치/판단불가 판정 |
| 설명 | 사람이 작성 | LLM 기반 판정 근거 설명 생성 |
| 확장성 | 사례별 처리 | End-to-End 시스템으로 확장 가능 |

### 3.3 Dataset & Preprocessing

#### 사용 데이터셋

- 조선일보 기사 데이터
  - 기간: 2025년 1월 1일 ~ 2025년 12월 31일
  - 수집 방식: Selenium 기반 웹 크롤링
  - 수집 대상: 기사 제목, 작성일, 섹션, URL, 기사 본문, 검색 키워드, 검색 구분 레이블
  - 검색 키워드: 국가데이터처, 통계청 등

- KOSIS 공식 통계 데이터
  - 통계목록 API
  - 통계자료 API
  - 메타정보 API
  - 통계설명 API

#### 데이터 특성

뉴스 기사 데이터는 자연어 문장으로 구성되어 있으며, 수치 표현과 통계 표현이 기사 문맥에 섞여 있다. 시점 표현이 “올해”, “지난해”처럼 상대적으로 등장하거나, 모집단이 앞문장에만 나오는 경우도 있다.

KOSIS 데이터는 구조화된 표 형태이며, 기관 ID, 통계표 ID, 항목 ID, 분류 코드, 단위, 수록 시점 등의 파라미터로 조회된다. 뉴스 표현과 KOSIS 항목명이 정확히 일치하지 않는 경우가 많다.

#### 전처리

뉴스 기사 전처리:

- HTML 태그 제거
- 기자명, 광고 문구, 저작권 문구 제거
- 공백 정리
- 문장 분리
- 숫자 및 단위 표현 탐지
- 앞뒤 문맥 저장
- 수치 포함 문장 후보 추출

KOSIS 데이터 전처리:

- API 응답 JSON 저장
- DataFrame 변환
- 통계표명, 단위, 시점, 분류 항목 추출
- `DT`, `PRD_DE`, `UNIT_NM`, `C1_NM`, `C2_NM` 등 주요 필드 정리
- 통계표별 검증 가능 주장 유형 정리

### 3.4 왜 효과가 있을 것으로 예상하는가

- **H1.** 뉴스 주장을 지표, 값, 단위, 시점, 모집단으로 구조화하면 공식 통계와의 매칭 가능성이 높아질 것이다.
- **H2.** KOSIS 통계표의 분류 항목과 메타정보를 함께 활용하면 단순 키워드 검색보다 정확한 통계표 후보를 찾을 수 있을 것이다.
- **H3.** 공식 API 조회와 표 연산을 결합하면 기사 수치의 일치 여부를 재현 가능한 방식으로 검증할 수 있을 것이다.
- **H4.** 판단불가 유형을 명시적으로 분류하면 시스템의 한계를 투명하게 설명할 수 있을 것이다.
- **H5.** LLM은 최종 판정 자체보다 판정 근거와 계산 과정을 자연어로 설명하는 데 효과적으로 활용될 수 있을 것이다.
